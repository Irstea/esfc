Principe général

PrototypePHP est un squelette d'application, qui s'appuie sur des classes éprouvées :
- ADODB pour l'accès aux bases de données (adodb.sourceforge.net)
- OBJETBDD pour gérer la lecture et l'enregistrement des données dans les tables (mapping relationnel-objet)
- PHPCAS (ou ESUP-PHPCAS) pour gérer l'identification via un SSO-CAS
- SMARTY pour l'affichage des pages html
- PHPGACL pour la gestion des droits.

Il est basé sur un modèle MVC (Modèle - vue - contrôleur). Une page unique interroge un fichier xml, qui décrit chaque module (chaque page appelée), en indiquant les droits nécessaires. Ce contrôleur vérifie les droits, exécute le module, enchaîne le cas échéant sur un autre module en fonction du code de retour, puis affiche la page html. Il génère également le menu automatiquement en fonction des paramètres indiqués dans le fichier XML.

PrototypePHP est distribué sous licence LGPL (hors modules externes, où les licences spécifiques s'appliquent) : les modifications doivent être reversées à la communauté.

Nouveautés

Version 0.4 du 20/08/2009

- intégration des dernières versions des classes ADODB (version 5.09.a), OBJETBDD (version 2.0), SMARTY (2.6.26).
- il est maintenant possible de déclarer l'attribut ajax dans le fichier param/actions.xml, ce qui permet d'inhiber l'affichage par défaut dans la page (indispensable pour les modules fonctionnant en mode ajax)
- récupération automatique de la langue du navigateur, pour orienter vers le bon fichier de traduction
- rajout de paramètres permettant de déclencher différents mode "debug", soit sur ADODB, soit sur OBJETBDD
- modification du cryptage des mots de passe (en cas d'identification via mot de passe) : le md5 est abandonné au profit du sha256. La structure de la base de données doit évoluer en conséquence (longueur de la zone passe de 32 Ã  64 caractères)
- ne fonctionne qu'avec la version 5 de PHP (intégration de la gestion des exceptions).
Modifications liées Ã  OBJETBDD :
- possibilité de vérifier les données saisies avant leur insertion dans la base de données (champ obligatoire, longueur des caractères, type des données, masque de saisie via une expression régulière)
- transcodage des caractères spéciaux html (< > & ") dans les deux sens (affichage et insertion en base de données)
