Gestion des droits

La gestion des droits s'effectue en deux temps. La première, c'est l'identification de l'utilisateur, qui peut être réalisée soit en base de données, soit en s'appuyant sur un annuaire LDAP, soit en utilisant un service unique d'identification (SSO/CAS).

A ce login vont être affectés des droits. Les droits sont gérés par l'application phpgacl, application qui permet également de gérer de manière centralisée les droits pour un ensemble d'applications. PhpGACL propose tout un tas de mécanismes pour gérer de façon très fine les droits.

Principe de PhpGACL

PhpGACL gère 2 types d'objets :
– ACO ( Access Control Objects ) : objets auxquels ont donne les droits.
– ARO ( Access Request Object ) : utilisateur qui souhaite accéder aux objets.
PhpGACL permet d'attribuer des droits positifs (accès autorisés) ou bien des droits
négatifs (accès refusés).
Les utilisateurs ARO peuvent faire partie de plusieurs groupes, qui auront des droits sur les objets ACO.

Se connecter à l'application

Quelque soit le mode d'identification qui est utilisé dans l'application (gestion par base de données, annuaire LDAP ou SSO), vous devez disposer du login admin, qui vous permettra de manipuler les droits. Si vous ne pouvez pas vous connecter avec le login admin, vous avez deux possiblités :
- vous basculez temporairement le mode de fonctionnement de l'identification en mode BDD, et vous identifiez avec le login admin (mot de passe par défaut : password).
- vous modifiez manuellement la table gaclaro, pour remplacer le login admin par le votre.
 Vous pourrez alors créer les logins qui vous intéressent dans l'application de gestion des droits, et basculer ensuite sur le mode d'identification qui vous convient.

Manipuler les droits

Lancez l'application, puis cliquez sur Paramétrage, puis Paramétrage>Gestion des droits.

Dans l'onglet [ACL admin], cliquer sur Login, puis, dans la zone "Access Request Objects", sur [Edit]. Renseigner alors tous les logins des personnes qui devront soit consulter les données, soit gérer l'application.

Dans l'onglet [ARO Group Admin], pour chaque ligne consult, gestion, admin, cliquer sur [Assign ARO], puis sélectionner les logins qui seront autorisés pour ce niveau (clic sur Login, sélection des logins, puis [>>]. N'oubliez pas de valider par [Submit].
