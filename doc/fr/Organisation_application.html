Organisation de l'application

Dossiers utilisés par prototypePHP

CSS

Contient les feuilles de style.

doc

Documentation de l'application, avec un sous-dossier par langue. La page index.php gère les accès à la documentation.

Identification

Modules et classes utilisées pour gérer les logins.

install

Documentation sur l'installation de l'application, scripts de génération de la base de données.

javascript

fonctions javascript intégrées aux pages html (vérification de mot de passe, calendrier...)

ldap

Contient une classe de gestion des accès aux annuaires ldap.

locales

dossier contenant les fichiers de langue. Un fichier par langue supportée.

menu

Contient le fichier qui génère le menu de l'application.

navigation

Contient d'une part le fichier actions.xml, qui décrit l'accès aux modules, et d'autre part, la classe qui permet de manipuler ce fichier.

param

Contient tous les fichiers de paramètres.

plugins

Contient tous les plugins "commerciaux" utilisés dans PrototypePHP : adodb, esup-phpcas, objetBDD, phpgacl, smarty.
A noter que phpcas-simple est un client CAS "léger", qui est conservé au cas où, mais qu'il est déconseillé d'utiliser aujourd'hui.

display/templates

C'est le dossier de stockage des pages HTML qui seront manipulées par SMARTY. Ce dossier contient autant de sous-dossiers que de sous-dossiers dans l'application, qui ont besoin de générer des pages html.

display/templates_c

Dossier utilisé par SMARTY quand on active la mise en cache. Dans ce cas, ce dossier doit être en écriture totale et être nettoyé régulièrement.

Les autres dossiers

Leur utilisation est facultative. Ils ne sont que des suggestions d'organisation de l'application...

administration

Tout ce qui touche à l'administration de l'application. En général, les modules déclarés ici ne doivent être accessibles qu'aux administrateurs.

gestion

C'est le dossier où l'on stockera les modules destinés à la gestion de l'application : modification des tables de paramètres...

images

Pour stocker les icones, images, fonds d'écrans...

test

Dossier destiné aux développeurs, quand il faut tester quelques aspects de l'application sans toucher aux modules initiaux...
